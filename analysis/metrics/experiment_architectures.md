## Общий каркас (backbone)

Все варианты используют один и тот же каркас **LSTM-CNN-CRF**, если не сказано иное.

**Извлекатель признаков LSTMCNN** (`src/models/lstm_cnn.py`, класс `LSTMCNN`):
1. `Dropout1d(p=0.1)` по каналам сырого эмбеддинга.
2. `Conv1d(C_in → 32 фильтра, kernel=3, padding=1)` + ReLU.
3. `Dropout1d(p=0.1)`.
4. Двунаправленный LSTM: вход=32, hidden=64 → выход 128 на позицию.
5. `Conv1d(128 → 64 фильтра, kernel=5, padding=2)` + ReLU.

Выход: `[B, L, 64]` — контекстные признаки на остаток.

**Голова эмиссий:** `Linear(64 → num_labels)`, где num_labels=3 (None / Peptide / Propeptide).

**Многосостоянийный CRF** (101 состояние для 3-классовых моделей):
- Состояние 0 = None; состояния 1–50 = прогресс пептида (строгая грамматика длины 5–50);
  состояния 51–100 = прогресс пропептида (та же грамматика).
- 3-классовый вектор эмиссий «размножается» в 101 состояние: логит пептида копируется во
  все пептидные состояния, логит пропептида — во все пропептидные.
- Обучается отрицательным лог-правдоподобием CRF (усреднение).
- Лучший чекпойнт = max по (F1_pep + F1_prop)/2 на валидации.

**Общие гиперпараметры** (если вариант явно не переопределяет): `epochs=100`,
`batch_size=48`, `lr=1e-4`, `optimizer=Adam`, `dropout=0.1`, `kernel_size=3`,
`num_filters=32`, `hidden_size=64`, `label_type=multistate_with_propeptides`.

> Как понимать раздел: для **простых замен эмбеддинга** (та же архитектура, другой вход)
> описание короткое — меняются только размерность и источник. Для **сложных архитектур**
> (телескопический сегментный CRF, boundary/bond, gated-проекторы) — подробное объяснение
> понятным языком: что конкретно происходит и зачем.

## Таблица 1: Архитектурные изменения

Все варианты Таблицы 1 используют эмбеддинги **ESM2** (`embedding_dim=1280`).

### T1-1 · ESM2 (базовая)
**Папка:** `runs/train_run_esm2` · **Класс:** `LSTMCNNCRF` (`lstmcnncrf`).
Дословно каркас выше: сырые ESM2 → LSTMCNN → linear → 101-состоянийный CRF; Adam lr=1e-4;
ни проектора, ни доп.сигналов, ни доп.лоссов.

### T1-2 · ESM2 + телескопический сегментный CRF
**Папка:** `runs/esm2_telescoping_segmental` · **Класс:** `LSTMCNNCRFTelescopingSegmental`
(`lstmcnncrf_telescoping_segmental`). **Это сложная архитектура — подробно ниже.**

**Проблема, которую решаем.** В обычном CRF одна и та же «грубая» эмиссия PEPTIDE
копируется во все 50 пептидных состояний `pep_1..pep_50`. То есть нейросеть говорит лишь
«этот остаток похож на пептид», но НЕ говорит «этот остаток похож на *начало* / *середину*
/ *конец* пептида». Состояний много, но осмысленного сигнала в них нет, и состояние
`pep_37` не имеет биологического смысла, пока неизвестна длина сегмента.

**Идея телескопической суммы (telescoping sum).** Мы хотим, чтобы накопленный вдоль пути
`pep_1 → pep_2 → … → pep_m` счёт был равен осмысленному счёту *всего* участка `[s, e]`
длины `m`, назовём его `SpanScore(s,e)`. Для этого эмиссии заменяются на **разностные
(delta)**:

```
эмиссия(i, pep_k) = SpanScore(i−k+1, i) − SpanScore(i−k+1, i−1)
```

При суммировании вдоль сегмента все промежуточные `SpanScore` взаимно сокращаются
(телескопируются), и в сумме остаётся ровно `SpanScore(s,e)`:

```
SpanScore(s,s)
+ [SpanScore(s,s+1) − SpanScore(s,s)]
+ [SpanScore(s,s+2) − SpanScore(s,s+1)]
+ … = SpanScore(s,e)
```

Главное преимущество: можно оставить **обычный** механизм forward/backward многосостоянийного
CRF без изменений, просто подставив другие значения эмиссий.

**Что такое SpanScore.** Для участка длины `m` каждый остаток `i` внутри него имеет
известную *относительную координату* `t_rel = (i−s)/(e−s)` (0 = начало, 1 = конец; для
длины 1 берётся 0.5). Модель отдельной головой (`position_head`: LayerNorm + Dropout +
Linear(64→2) → sigmoid, инициализирована нулём → старт ≈ 0.5) предсказывает для каждого
остатка *предпочтительную* относительную позицию `r_pred`. Счёт участка:

```
SpanScore(s,e) = Σ грубых_логитов_по_остаткам
               + scale · Σ позиционный_счёт(r_pred[i], t_rel[i])
позиционный_счёт = −|t_rel − r_pred| / τ   (это лог-потенциал, не вероятность)
```

То есть участок получает бонус, когда остатки реально стоят на тех относительных позициях,
которые им «к лицу». В этом запуске: `position_score_mode=neg_abs`, `τ=0.25`, `scale=0.25`,
вспомогательный позиционный лосс выключен (`relative_position_loss_lambda=0`),
`segmental_max_len=50`. Декодированные строгие пути конвертируются обратно в legacy-состояния,
чтобы метрики считались без изменений.

> Примечание: в `texs/brainstorm/segmental_crf_relative_position_spec.md` описан более
> общий полу-марковский (semi-Markov) сегментный CRF с перечислением спанов и базис-функциями
> фаз. В коде реализован именно **телескопический** вариант (delta-эмиссии поверх обычного
> CRF) — он проще и не требует отдельного DP по спанам.

### T1-3 · ESM2 + Aho-слияние на эмиссиях
**Папка:** `runs/esm2_aho_emission_fusion` · **Класс:** `LSTMCNNCRFAhoEmissionFusion`.
Вход — конкатенация `[ESM2 | Aho]` (1280+76=1356). Ветка последовательности: ESM2 →
LSTMCNN → `Linear(64→3)` базовые эмиссии. Ветка Aho (позднее слияние): признаки Aho (76-d,
разреженные попадания словаря) → `LayerNorm + Dropout + Linear(76→3)` → добавочное
смещение на класс, инициализировано нулём (старт = базовая модель). `aho_hidden_size=0`
(прямой linear). Итоговая эмиссия = база + aho_bias.

### T1-4 · ESM2 + (Aho → скрытый слой 32) слияние на эмиссиях
**Папка:** `runs/esm2_aho_emission_fusion_h32`. Как T1-3, но ветка Aho — двуслойный MLP
`Linear(76→32) + GELU + Linear(32→3)` (последний слой обнулён).

### T1-5 · ESM2 + Aho-слияние на скрытом состоянии
**Папка:** `runs/esm2_aho_mid_fusion_raw_m64` · **Класс:** `LSTMCNNCRFAhoMidFusion`.
ESM2 → LSTMCNN даёт `h_i` (64-d). Aho-признаки (LayerNorm) дают `a_i` (76-d). Mid-fusion
MLP по `[h_i; a_i]` (140-d, hidden=64) добавляет коррекцию эмиссий (обнулён старт). То есть
коррекция Aho обусловлена и нейроконтекстом, и попаданиями словаря.

### T1-6 · ESM2 + Aho-слияние на скрытом состоянии (только пептиды)
**Папка:** `runs/esm2_aho_mid_fusion_raw_m64_pep_only`. Как T1-5, но коррекция Aho
применяется только к классу Peptide (`aho_none_scale=0`, `aho_pep_scale=1`,
`aho_propep_scale=0`).

### T1-7 · ESM2 + Aho-сигнал добавляется к переходам/состояниям CRF
**Папка:** `runs/esm2_aho_transition_bias_sparse_trainable_zero` · **Класс:**
`lstmcnncrf_aho_transition_bias_sparse` — **этого класса больше нет в коде**, чекпойнт не
грузится (`strict=True` падает) → MCC/AUC = N/A. По чекпойнту: Aho-смещения добавлялись
прямо к эмиссиям состояний CRF (start/inside/end каждой ветки), веса обучаемые,
инициализированы нулём.

### T1-8 · ESM2 + Aho раннее слияние (concat)
**Папка:** `runs/esm2_aho_tribranch` · **Класс:** `LSTMCNNCRFTriBranchResidual`.
Трёхветочный gated-residual проектор ДО LSTMCNN: ESM2 (1280→256, главная ветка) + Aho
(76→16, gated residual, масштаб 0.05, смещение гейта −2.5). 256-d слитое представление →
LSTMCNN. Отличие от T1-3/T1-5: Aho вмешивается *до* LSTM/CNN, а не после.

### T1-9 · ESM2 + доп. лосс разрезов к ближайшей границе
**Папка:** `runs/esm2_bond_loss_soft_l005_w5_tau15` · **Класс:** старый `lstmcnncrf_bond_loss`
— **в коде нет** (сейчас `lstmcnncrf_boundary_bond_loss`), чекпойнт невосстановим → MCC/AUC
= N/A. По чекпойнту: базовый ESM2 LSTMCNN-CRF + ОДНА вспомогательная голова `bond_head`
(3-слойный MLP), предсказывающая логит разреза между соседними остатками; обучается мягким
лоссом (мишени размазаны вокруг истинных сайтов разреза экспонентой, окно 5, τ 1.5, вес
0.05). Голова влияет ТОЛЬКО на лосс обучения, не на эмиссии на инференсе — то есть это
базовая модель + доп.лосс. (Boundary-головы здесь НЕТ — это отличие от текущего класса.)

### T1-10 · ESM2 с AdamW

## Лучшая комбинированная конфигурация — ESM-C 6B × boundary/bond

**Папка:** `runs/esmc6b_boundary_bond` · **Класс:** `LSTMCNNCRFBoundaryBondLoss`. Это
осознанный кросс «лучший эмбеддинг × дополняющая архитектура», а не строка sweep'а.

**Что делает boundary/bond (подробно).** В отличие от базовой модели, здесь добавлены два
компонента:
1. **Boundary-голова состояний** (`BoundaryStateEmissionHead`): из контекстных признаков
   предсказывает 6 логитов — начало/внутри/конец для пептида и для пропептида — и
   **добавляет их прямо к эмиссиям состояний CRF**: «внутри» прибавляется ко всем
   состояниям ветки, «начало» — к первому состоянию, «конец» — к последнему. Инициализирована
   нулём (старт = база). Именно эта голова заостряет границы разрезов и используется на
   инференсе.
2. **Вспомогательный мягкий bond-лосс** (`BondBoundaryHead`): предсказывает логит разреза
   между соседними остатками, обучается мягкими мишенями вокруг истинных сайтов (окно 5,
   τ 1.5, λ=0.02, pos-weight 10). Влияет только на обучение.

**Почему сработало.** ESM-C 6B сам по себе — высокий recall / низкая precision. Boundary-голова
конвертирует обильный-но-размытый по-остаточный сигнал в чёткие ±3-вызовы → precision +0.14
(0.570→0.714) при сохранении recall. На уже-сбалансированной ESM2 та же архитектура почти
не помогла (F1 0.606 ≈ 0.607) — польза зависит от P/R-профиля эмбеддинга.

**Результат (TEST, fp32, drift 0.000):** F1 0.657 / P 0.714 / R 0.609 / MCC 0.765 —
лучшие F1 и MCC в проекте. (Подробности и однозначность по метрике — `texs/error_analysis/combine_best.md`.)

Та же трёхветочная gated-residual архитектура, что T1-8, но на эмбеддинге ESM2+ (1290-d, с
10 биохим. признаками).

### ESM2 на датасете uniprot_2026
**Папка:** `runs/uni2026_run_esm2` · **Класс:** `LSTMCNNCRF`. Базовая архитектура, но на
более новом, НЕ полностью препроцессированном датасете uniprot_2026. Не сопоставима
напрямую с запусками на uniprot_2022.

### Серия масштабирования данных
**Папки:** `runs/scaling_trainfrac{50..90}`, `runs/train_run_esm2_100` (исправленная
серия) и `runs/train_run_esm2_{25,50,75}` (старая, ненадёжная серия). Все — базовая
`LSTMCNNCRF` на ESM2; меняется только доля обучающих кластеров (0,1,2). Подробно —
`texs/error_analysis/data_scaling.md`. (Старая серия сабсэмплила все кластеры, включая
valid/test → несопоставима; `train_run_esm2_25` к тому же указывал на файл 50%.)

## Таксономия классов моделей

| Ключ модели | Описание |
|:--- |:--- |
| `lstmcnncrf` | Сырые эмбеддинги → LSTMCNN → CRF |
| `lstmcnncrf_telescoping_segmental` | LSTMCNN + телескопический сегментный CRF (delta-эмиссии) |
| `lstmcnncrf_boundary_bond_loss` | LSTMCNN + boundary-голова состояний + вспом. bond-лосс |
| `lstmcnncrf_aho_emission_fusion` | LSTMCNN + позднее слияние Aho (смещение эмиссий) |
| `lstmcnncrf_aho_mid_fusion` | LSTMCNN + слияние Aho на скрытом состоянии |
| `lstmcnncrf_aho_state_bias` | LSTMCNN + смещение состояний CRF признаками Aho |
| `lstmcnncrf_tribranchresidual` | Трёхветочный gated-проектор (ESM2+Aho/+) → LSTMCNN |
| `lstmcnncrf_projector_split` | Раздельные проекторы (ESM2 + 3Di) → concat → LSTMCNNCRF |
| `lstmcnncrf_gated3diresidual[_conv]` | ESM2 (главная) + структурная ветка (gated residual [+Conv1d]) |
| `lstmcnncrf_projector_multiscale` | Многомасштабный свёрточный проектор перед LSTMCNN |
| `lstmcnncrf_esm2_lora` | LoRA-дообучение ESM2 онлайн + голова |
| `lstmcnncrf_aho_transition_bias_sparse` | (утрачен) смещения Aho в состояния CRF |
| `lstmcnncrf_bond_loss` | (утрачен, старый) базовая + bond-лосс без boundary-головы |
