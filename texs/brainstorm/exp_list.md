Идеи на основании выводов (по приоритету):

(a) [аналитика, без GPU] Проверить новизну ревизии 2026: прогнать peptide_similarity
    (тест-пептиды 2026 против train-пептидов 2022). Если добавляет НОВЫЕ семейства
    (хвост <0.30) — обучаться стоит; если те же семейства +14% объёма — эффект ~как
    +10% данных на кривой масштабирования (~+0.014 F1). Отвечает «стоит ли тратить
    часы на обучение на 2026» ДО обучения.

(b) [данные] Целевой набор недопокрытых семейств. У нас уже 49k внешних AMP-DB
    пептидов в AHO-словаре, и внешние попадания реально помогают (+0.082, aho_prior).
    Затащить те, у кого есть полный белок-предшественник с разметкой разрезов
    (DRAMP/APD линкуются на UniProt), как обучающие примеры → закрыть хвост <0.30.
    Бьёт прямо в bottleneck (data_need.md). Камень: нужен предшественник, не только
    зрелый пептид.

(c) [без новых данных] Reweighting/oversampling по редкости семейства: поднять вес
    лосса (или оверсэмплить) для пептидов из низкопокрытых бакетов похожести. Прямо по
    выводу про покрытие, один прогон. Лучшее «дёшево/обосновано».

(d) [дешёвый плюс] Gated AHO (follow-up из aho_prior.md): применять приор только где
    pep.inside>0 → сохранить +0.08 на 20% попаданий, убрать −0.055 шума на 80%.
    Можно поверх победителя ESM-C6B+boundary.

(e) [очередь] ESM-C6B ⊕ 3Di + boundary/bond: 3Di лучше всех на крайних длинах
    (len 5 и 31–50, где победитель слабее) — тест на аддитивность (structural_potential.md).

(f) [гигиена] 3 сида для победителя и baseline: сейчас +0.05 ≈ 2.5× шума одного сида;
    усреднение по 3 сидам делает headline-вывод железобетонным.
