Целевой просмотр высокориск. кода (vibe-coded `infer.py` + метрики + конверсии), по следам

## НАЙДЕНО И ИСПРАВЛЕНО

### 🔴 Residue ROC-AUC считался по вырожденному score (commit 88bd820)
- **Где:** `infer.py::get_task_scores`.
- **Суть:** CRF возвращает поостаточные маргинали по ВСЕМ 101 состоянию (None + 50
  пептидных + 50 пропептидных). `get_task_scores` же трактовал массив как 3-классовый
  coarse-вектор и брал **столбец 1** (= P(состояние 1) = только START-остаток пептида)
  как score пептида и **столбец 2** (= P(состояние 2), тоже ПЕПТИДНОЕ состояние!) как
  score пропептида.
- **Следствие:** residue ROC-AUC во всех `*_infer.json` и во всех AUC-ячейках таблиц —
  мусор. Smoking gun: пропептидный AUC ≈0.5, местами <0.5 (хуже случайного) у многих
  моделей.
- **Почему MCC был прав, а AUC нет:** MCC берёт декодированный путь (состояния 1..50 via
  `state_in_ranges`) — корректно. Только AUC зависел от сломанного `y_score`. Поэтому MCC
  трекал F1, а AUC «инвертировал».
- **Фикс:** при >3 столбцах (маргинали) score = сумма маргиналей по диапазону состояний
  задачи (P(остаток ∈ пептидных состояниях)). Проверка на baseline `train_run_esm2`:
  пропептидный AUC **0.503→0.963**, пептидный **0.860→0.969**, MCC без изменений
  (0.694/0.745/0.750). Столбцы AUC регенерируются полным fp32-инференсом + пересбор таблиц.
- **Вывод для отбора моделей не меняется:** F1+MCC остаются основными (см.
  `metrics_choice.md`); AUC теперь хотя бы корректен как диагностика.

## ПРОВЕРЕНО — ОК (ложные тревоги / не баги)

- `compute_residue_level_metrics` (infer.py): `true_seq[:seq_len]` обрезает истинные метки
  до длины декода — корректно снимает паддинг (Viterbi с маской возвращает истинную длину).
- `state_in_ranges` + константы состояний: пептид (1,50), пропептид (51,100), инклюзивно —
  верно. Residue MCC корректен.
- `convert_path_to_peptide_borders`: открытие на start_state(1), закрытие на stop_state(50)
  — соответствует грамматике (каждый пептид принудительно завершается в состоянии 50).
- `compute_all_metrics`: offset=1 (1-based), окна ±w работают (готово для свипа #12);
  «all» корректно объединяет pep+propep.
- `get_counts_for_protein`: подтверждён ТОЛЬКО известный shadowing-баг (двойной отчёт
  old/new, см. `methodology.md`); `break`+отсутствие удаления matched-pred теоретически
  даёт переиспользование одного pred, но при ±3 и группировке перекрытий эффект мал.
- `_add_boundary_state_emissions` (победитель): добавление inside→все состояния ветки,
  start→первое, end→последнее — корректно, согласовано с грамматикой.
